📋 WHAT EACH FILE DOES
════════════════════════════════════════════════════════════════

CSS/variables.css
  • 30+ CSS custom properties (--lime, --amber, --bg-base, etc.)
  • @keyframes animations (fadeUp, arcPulse, blink, etc.)
  • Global box-sizing and resets
  → Change here to restyle ENTIRE dashboard

CSS/layout.css
  • Navigation bar (.topnav) - 52px glassmorphic header
  • Main grid (.main) - 3 columns (280px | 1fr | 320px)
  • Status bar (.bottom-strip) - 60px footer
  • Column scrolling and custom scrollbars
  → Adjust layout proportions and navigation

CSS/components.css
  • .card - Base card styling with glassmorphism
  • .gauge-container - SVG tachometer container
  • .fuel-card, .health-card - Specialized cards
  • Status badges, metrics, telemetry grid
  • Responsive media queries (1920px, 1600px)
  → Style individual UI components

JS/gauge.js
  • class GaugeAnimator - Manages SVG needle animation
  • setRPM() - Set target RPM for smooth animation
  • calculateEfficiencyZones() - Compute ECO/OPTIMAL RPM
  • updateZoneLines() - Visual zone indicators
  → Controls tachometer needle and efficiency lines

JS/dashboard.js
  • class Dashboard - Main application logic
  • updateTelemetry() - Update all metrics with data
  • calculateEfficiency() - Engine efficiency percentage
  • setupTerrainAngleUpdater() - Pitch/roll display
  • startDemoMode() - Simulate telemetry for testing
  → Processes data and updates UI

JS/telemetry-client.js
  • class TelemetryClient - WebSocket management
  • Handles Socket.io connection + auto-reconnection
  • normalizeTelemetryData() - Standardize data format
  • Connection status UI updates
  • Graceful fallback to demo mode if disconnected
  → Receives real-time telemetry from backend

JS/ui-updater.js
  • class UIUpdater - Advanced UI rendering
  • Card hover effects and glow indicators
  • Notifications (info/warning/error)
  • Theme switching (dark/light)
  • Keyboard shortcuts (F11, Alt+T)
  • Interactive element setup
  → Adds polish and interactivity to UI

HTML/index.html
  • CLEAN semantic markup ONLY
  • No inline CSS (<style> tags removed)
  • No inline JavaScript (moved to modules)
  • Proper <link> for CSS imports
  • Proper <script> tag loading order
  • Socket.io integration ready
  → Foundation for all styling and logic

🔧 HOW TO CUSTOMIZE
════════════════════════════════════════════════════════════════

Change Dashboard Colors:
  1. Open: Dashboard/frontend/css/variables.css
  2. Edit: --lime, --amber, --red, --teal properties
  3. Refresh browser (Ctrl+R)
  ✅ All cards update instantly

Adjust Layout Width:
  1. Open: Dashboard/frontend/css/layout.css
  2. Edit: grid-template-columns value
  3. Refresh browser
  ✅ Columns resize dynamically

Modify Engine Data:
  1. Open: Dashboard/frontend/js/dashboard.js
  2. Edit: tractorWeight, implementWeight, maxRPM
  3. Refresh and test demo mode
  ✅ All calculations update

Change Efficiency Zones:
  1. Open: Dashboard/frontend/js/gauge.js
  2. Edit: baseEcoRPM, baseOptimalRPM
  3. Test with: window.gaugeAnimator.calculateEfficiencyZones(400, 18, 1.0)
  ✅ Zones recalculate automatically
